# Release v12.0.0 — sndr-platform multi-engine architecture

**Release date**: TBD (after acceptance criteria pass)
**Status**: in-progress
**Breaking changes**: yes — see Migration section below

## Highlights

This is the first release under the **sndr-platform** name. The codebase
moves out of `vllm/sndr_core/` to a top-level `sndr/` package and gains the
foundations for multi-engine support (vLLM today, SGLang next).

### What's new

- **Multi-engine architecture**: `EngineAdapter` ABC accommodates vLLM and
  the SGLang skeleton without code changes in the orchestrator layers.
- **Layered architecture**: kernel → engines → dispatcher → apply →
  product_api → cli. Enforced by `tools/ci/verify_layer_imports.py`.
- **Per-pin manifests**: YAML files in `sndr/engines/<engine>/pins/<pin>/`
  describe upstream file md5s and anchor snippets. Drift detection compares
  live installs against these manifests.
- **Drift detection**: `tools/drift_check.py` + daily CI cron in
  `.github/workflows/drift_check.yml`.
- **Commercial tier plumbing**: `sndr/plugins/loader.py` discovers patches
  registered by external wheels (e.g. `sndr-engine`) via setuptools entry
  points. License verification gates engine-tier patches.
- **`sndr` CLI**: `sndr engines.list`, `sndr engines.info`, `sndr pins.list`,
  `sndr health` — JSON or text output.
- **FastAPI control plane**: `/api/v1/engines`, `/api/v1/engines/{engine}/pins`,
  `/api/v1/licensing/status`, `/api/v1/health`. Auto-generates OpenAPI for
  GUI TypeScript types.
- **GUI**: Carbon Design System (g100 theme), Lingui i18n (en + ru), Zustand
  state stores, TanStack Query API client.
- **309 patches migrated** to `sndr/engines/vllm/patches/` (24 families).
- **Tools**: `tools/migrate_patches.py`, `tools/migrate_module.py`,
  `tools/manifest_gen.py`, `tools/drift_check.py`.

## Breaking changes

### Import paths

| Old | New |
|---|---|
| `from vllm.sndr_core import X` | `from sndr import X` |
| `from vllm.sndr_core.dispatcher.registry import PATCH_REGISTRY` | `from sndr.dispatcher.registry import PATCH_REGISTRY` |
| `from vllm.sndr_core.core import TextPatcher` | `from sndr.kernel import TextPatcher` |
| `from vllm.sndr_core.detection import gpu_arch_profile` | `from sndr.detection import gpu_arch_profile` |
| `from vllm.sndr_core.integrations.attention.X import Y` | `from sndr.engines.vllm.patches.attention.X import Y` |

**Backward compatibility shims** are in place for all old import paths.
They emit a `DeprecationWarning` at import time and continue to work for
the entire v12.x cycle. They will be removed in v13.0.

### Container mount path

Old:

```
-v /path/to/genesis-vllm-patches/vllm/sndr_core:/usr/local/lib/python3.12/dist-packages/vllm/sndr_core:ro
```

New:

```
-v /path/to/genesis-vllm-patches/sndr:/usr/local/lib/python3.12/dist-packages/sndr:ro
-e SNDR_ENGINE=vllm
```

Old launchers continue to work via shims for v12.x. A universal launcher
template is documented in `docs/guides/PIN_UPGRADE.md`.

### Environment variables

New:

- `SNDR_ENGINE` (default: `vllm`) — engine adapter to load
- `SNDR_ENGINE_PIN` (default: auto-detect) — manifest pin to target
- `SNDR_CONFIG` — path to YAML override
- `SNDR_HOME` (default: `~/.sndr`) — state directory
- `SNDR_LOG_LEVEL`, `SNDR_STRICT_DRIFT`, `SNDR_STRICT_APPLY`

Existing `GENESIS_ENABLE_*` env vars continue to work unchanged.

## Migration guide

### For operators

1. Update launcher scripts to mount `sndr/` instead of `vllm/sndr_core/`.
2. Add `-e SNDR_ENGINE=vllm` to docker run command.
3. (Optional) Migrate from per-patch env vars to a YAML config via
   `SNDR_CONFIG`.

### For downstream consumers

1. Update imports to use `sndr.*` paths.
2. Continue using `vllm.sndr_core.*` paths during v12.x — they still work
   but emit deprecation warnings.
3. By v13.0, complete the migration; the shims will be removed.

### For patch authors

1. New patches go in `sndr/engines/vllm/patches/<family>/<patch_id>.py`.
2. Add an entry to `sndr/dispatcher/registry.py` (the single
   patch-registry dict).
3. Follow naming conventions in `docs/guides/PATCH_AUTHORING.md`.

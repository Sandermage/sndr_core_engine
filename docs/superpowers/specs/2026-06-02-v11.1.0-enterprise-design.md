## 7. Track C — P3.3 PersistentBufferRegistry

### C.1 Module shape

NEW file: `vllm/sndr_core/runtime/persistent_buffer_registry.py`

```python
"""Operator-visible unified buffer pool registry.

Master plan §P3.3: 4 separate buffer pool managers (P36/PN12/P46/P39a)
share a common pattern — allocate-once, reuse-forever buffers in vLLM hot
paths. This module centralizes the lookup surface so:

  - Operators can `sndr patches show buffer_registry` and see all pools
    + sizes + per-pool stats in one place.
  - Future pools migrate via single API (~10 LOC each).
  - Each pool's allocation logic stays IDENTICAL (byte-equivalent).

Public surface: PersistentBufferRegistry (singleton), BufferPool.

NOT a refactor of the pool LOGIC. Each pool's internal torch.empty() call
is unchanged. Only the LOOKUP SURFACE is unified.
"""
from __future__ import annotations
from threading import RLock
from typing import Optional, TYPE_CHECKING
if TYPE_CHECKING:
    import torch


class BufferPool:
    """One named buffer pool.

    Pools are pre-allocated by their patch module on first need. Acquire
    returns a tensor; release marks it reusable. Same allocation logic
    as each patch had pre-migration.
    """
    def __init__(self, name: str, max_size: int = 64):
        self.name = name
        self._buffers: list["torch.Tensor"] = []
        self._lock = RLock()
        self._stats = {"acquires": 0, "releases": 0, "allocations": 0, "reuses": 0}
        self._max_size = max_size

    def acquire(self, shape, dtype, device) -> "torch.Tensor":
        ...  # Returns matching buffer if available; else allocates fresh

    def release(self, tensor: "torch.Tensor") -> None:
        ...  # Marks tensor for reuse

    def stats(self) -> dict:
        return dict(self._stats)


class PersistentBufferRegistry:
    """Singleton pool index."""
    _instance: Optional["PersistentBufferRegistry"] = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._pools = {}
            cls._instance._lock = RLock()
        return cls._instance

    def get_pool(self, name: str, max_size: int = 64) -> BufferPool:
        with self._lock:
            if name not in self._pools:
                self._pools[name] = BufferPool(name, max_size=max_size)
            return self._pools[name]

    def all_pools(self) -> dict[str, BufferPool]:
        with self._lock:
            return dict(self._pools)


# Pool name constants — referenced by migrated patches + sndr CLI
POOL_TQ_DECODE_SHARED = "tq_decode_shared"        # P36
POOL_FFN_INTERMEDIATE_SCRATCH = "ffn_intermediate_scratch"  # PN12
POOL_GDN_GATING = "gdn_gating"                    # P46
POOL_FLA_KKT_PERSISTENT_A = "fla_kkt_persistent_a"  # P39a
```

### C.2 Migration pattern (per pool)

Each of P36/PN12/P46/P39a gets a thin wrapper. Pattern:

```python
# Before (P36, original)
def _ensure_tq_decode_buffer(layer, shape, dtype):
    if not hasattr(layer, "_tq_decode_buffer"):
        layer._tq_decode_buffer = torch.empty(shape, dtype=dtype, device="cuda")
    return layer._tq_decode_buffer

# After (P36, migrated thin wrapper)
def _ensure_tq_decode_buffer(layer, shape, dtype):
    from vllm.sndr_core.runtime.persistent_buffer_registry import (
        PersistentBufferRegistry, POOL_TQ_DECODE_SHARED,
    )
    pool = PersistentBufferRegistry().get_pool(POOL_TQ_DECODE_SHARED)
    return pool.acquire(shape, dtype, "cuda")
```

Same `torch.empty()` call happens under `pool.acquire()` internally. Byte-equivalent semantics — the operator-visible behavior (tensor shape, dtype, lifetime, idempotency) is identical.

### C.3 Migrations in scope

1. **P36** (TurboQuant shared decode buffers) — `vllm/sndr_core/integrations/attention/turboquant/p36_*.py`
2. **PN12** (FFN intermediate scratch pool) — `vllm/sndr_core/integrations/memory/pn12_*.py`
3. **P46** (GDN gating buffer pool) — `vllm/sndr_core/integrations/attention/gdn/p46_*.py`
4. **P39a** (FLA chunk_scaled_dot_kkt persistent A pool) — `vllm/sndr_core/integrations/attention/gdn/p39a_fla_kkt_buffer.py`

Each migration is a ~10 LOC patch — replace the existing `_ensure_*_buffer()` helper to delegate to the registry. The patch's `apply()` text-patch logic stays unchanged (we're only changing the post-patch helper code, not the upstream-replaced code).

### C.4 Risk mitigation without long bench

**Pre-migration verification per pool**:

1. **Unit test**: `tests/unit/runtime/test_persistent_buffer_registry.py`:
   - Singleton identity (`Registry()` is `Registry()`)
   - `get_pool()` returns the same `BufferPool` instance for same name
   - `BufferPool.acquire()` returns tensor with requested shape/dtype/device
   - Thread safety (parallel acquires don't race)
   - Stats incrementing correctly
2. **Unit test per migrated patch**: shape/dtype assertion that pool-returned tensor matches what the old `torch.empty()` would have returned
3. **Boot smoke** on rig:
   - Container boots, `/health` 200
   - Boot log shows all 4 patches applied (with new registry-backed paths)
   - `sndr patches show buffer_registry` lists 4 pools (sanity: each pool initialized lazily so it might be 0 until first request — that's OK)
   - 1 chat completion request → pools populate
4. **No long bench** — user accepted this tradeoff. Documented in CHANGELOG.

**Rollback if smoke fails**: each migration is one-file change; `git revert` per file restores prior behavior.

## Executive summary

Single-stream sits at 208 wall_TPS / 4.66ms TPOT / 162ms TTFT, decode dominated ~73% by memory-bandwidth-bound GDN/Mamba and ~27% by attention, with MTP K=3 acceptance (~0.78-0.81) as the largest TPS multiplier via TPS≈base×(1+accepted/K). The single biggest verified lever is MG-1: PN296 is mis-gated OFF in the hardware YAML, so the ENABLED PN299B/C/D/E num_warps-cap patches silently no-op (they read GENESIS_TRITON_AUTOTUNE_MAX_WARPS which only PN296 sets) — un-capped num_warps=8/16 configs spill on the 100KB/SM A5000; adding three YAML flags (PN296/PN298/PN299 in order) is config-only, correctness-neutral, and realistically +2-4% wall_TPS plus -5..-10ms TTFT. TA-2 (cap the un-autotuned num_stages=3 on fused_sigmoid_gating — the GDN decode kernel that fires on all 30 linear layers every step — to 2 on SM8.6) is the highest-leverage NEW kernel change for TPOT (-1..-3%, ~+2..+6 TPS) because it directly hits the 73% slice, but needs a one-shot timing A/B since direction is unverified. The acceptance levers — PN369 relaxed-accept (thin marginal band over the live P82=0.3 stack, ~+1-3%), MTP-4 P82 sweep (realistic +1-2% at iso-quality), and the harness-gated P67 fp16 PV-dot flip (~+1.5-3% wall, regression history v777) — are the realistic path past 228 toward the math-peak 246, but every one is BIASED or precision-sensitive and MUST pass genesis_quality_harness + tool-call/grammar regression before promotion. Quick wins that need no harness: MG-5 (PN368 Marlin MoE atomic-add wire, sub-percent but free), MT-2 (correct the FALSE "PROD draft=probabilistic" registry notes — pure correctness, prevents a -5.9% TPS footgun), PN347 retire (kill a real recurring false drift-alarm on an enabled quant patch), and the TA-3/TA-5 documentation/dormant-verification items. The realistic path to 228: stack MG-1 (+2-4%) + TA-2 (+1-3%) + MG-3 PN390 LSE (-1-2% TPOT) + PN369/MTP-4 acceptance (+1-3%) — these are additive across different bottlenecks (Triton occupancy, GDN pipeline, sampler HBM, accept-rate). Pushing toward 246 (the math-prompt peak) requires the harness-gated P67 fp16 flip and a winning P82/PN369 acceptance bump, both of which can invert via the (1+accepted/K) multiplier if acceptance drops, so they are gated not banked. TTFT improvement comes mainly from MG-1's smaller autotune search and TA-1's warp caps; TPOT from TA-2 + PN390 + the warp caps on per-token kernels (PN299C/E). The required first move is re-anchoring the two drift-skipped patches (PN133 reanchor as observability-only, PN347 retire-not-reanchor) to restore an auditable apply report — neither moves steady-state TPS but both unblock a clean baseline. Critically, MG-1 must precede the acceptance/precision A/Bs so each later lever is measured on a non-spilling baseline.

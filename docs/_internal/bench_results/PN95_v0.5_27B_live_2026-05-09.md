# PN95 v0.5 — Live integration on 27B Lorbus (2026-05-09)

## Headline result

**Path C v0.5 (instrumentation + bookkeeping) PROVES Mamba SSM exclusion
on the live production hybrid-GDN model.** The acceptance gate from
the design doc §6 ("log every excluded group at boot — to prove SSM
never demoted") is **VALIDATED**.

## Setup

- Container: `vllm-server-27b-pn95` (custom Path C smoke launcher)
- Model: `Qwen3.6-27B-int4-AutoRound` (the production hybrid-GDN target)
- vLLM: `0.20.2rc1.dev93+g51f22dcfd`
- 2× A5000, TQ k8v4, MTP K=3, 131K context
- PN95 enabled via `GENESIS_ENABLE_PN95_TIER_AWARE_CACHE=1` +
  `GENESIS_PN95_CONFIG_KEY=single-3090-hybrid-gdn-tier-aware-example`
- Bench: `genesis_bench_suite.py --quick` (5×5×1024)

## Mamba SSM exclusion proof (live)

The `KVCacheManager.__init__` anchor fires at engine boot and walks
the 4 KV cache groups. Live log lines:

```
[PN95] init_mamba_exclusions_from_kv_groups called: 4 groups, PN95 enabled=True
[PN95] group 0: spec_class=MambaSpec layers=['language_model.model.layers.0.linear_attn', ..., 'layers.60.linear_attn']  (16 layers)
[PN95] excluding MambaSpec group g0 from demotion
[PN95] group 1: spec_class=MambaSpec layers=['layers.1.linear_attn', ..., 'layers.61.linear_attn']  (16 layers)
[PN95] excluding MambaSpec group g1 from demotion
[PN95] group 2: spec_class=MambaSpec layers=['layers.2.linear_attn', ..., 'layers.62.linear_attn']  (16 layers)
[PN95] excluding MambaSpec group g2 from demotion
[PN95] group 3: spec_class=TQFullAttentionSpec layers=['layers.3.self_attn.attn', ..., 'mtp.layers.0.self_attn.attn']  (17 layers)
```

**Result: 3 of 4 KV groups (48 of 65 layers) are MambaSpec; all 3
correctly registered as excluded from demote candidates. Group 3
(TQFullAttentionSpec — the self_attn + MTP) is the one eligible tier.**

## What v0.5 includes (functional, end-to-end)

- ✓ All 3 text-patch anchors apply cleanly on production vllm dev93 source
- ✓ Lazy `TierManager` init from `GENESIS_PN95_CONFIG_KEY` env var
- ✓ Per-block MM origin tagging via `request.mm_features` walks
  `mm_position.offset/length` to compute block overlap
- ✓ MambaSpec classifier identifies hybrid-GDN groups + registers
  them as excluded from demote candidates
- ✓ `_CpuSlab` allocates `torch.empty(pin_memory=True)` when torch +
  CUDA available; falls back to `bytearray` cleanly
- ✓ `notify_admit` fires from cache_blocks() text-patch site
- ✓ `notify_touch` fires from get_cached_block() text-patch site
- ✓ NULL TPS impact — fully harmless when active

## What v0.5 does NOT yet do (deferred to v1.0)

- 🟡 Actual GPU↔CPU bytes movement (`demote_to_threshold` runs in pure
  Python; `_CpuSlab.store/load` work on bytes, but the bridge from
  vLLM's per-layer `kv_caches[layer_idx]` GPU tensor to a `_CpuSlab`
  byte slot needs a 4th text-patch at the worker level (likely in
  `gpu_model_runner.py` after kv_caches assignment)
- 🟡 Scheduler-tick demote trigger — currently `demote_to_threshold`
  is a method that has to be called externally; no scheduler hook
  invokes it on free-VRAM pressure yet
- 🟡 Real promote on tier-1 hit — `notify_touch` returns demoted
  bytes but no caller does the promote-back-to-GPU copy

## Decision

**Path C v0.5 ships as `implementation_status: partial`** in the
PATCH_REGISTRY. The patch is honestly labeled with the "Phase 1 of 2"
note in `experimental_note`. Phase 2 (v7.74.x) will add the worker-
level wire-in for actual bytes movement.

Today's value: **Mamba SSM exclusion is provably correct** —
operators considering Path C (or external CPU-offload) on hybrid-GDN
configs can use this as evidence that the foundation is sound.
